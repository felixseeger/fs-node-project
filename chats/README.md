# FS Node Project: Chat Storage Structure

This document defines the filesystem-based representation of user chat sessions. The structure allows chat data, typically stored in Firestore (`ai_chats` collection), to be exported, archived, or read directly by the Chat UI or AI agents.

## Directory Hierarchy

All chat data is organized under the `chats/` directory at the project root. The hierarchy is defined by the user ID and the specific chat session ID.

```text
chats/
└── {userId}/                   # Unique identifier for the user
    └── {chatId}/               # Unique identifier for the chat session
        ├── index.md            # The full conversation in human-readable Markdown
        ├── meta.json           # Structured metadata about the chat session
        └── attachments/        # Directory containing assets (images, audio, video)
            ├── img-123.png
            └── vid-456.mp4
```

## File Schemas

### `meta.json`

The `meta.json` file contains structured data about the chat session, making it easy to index, search, and reconstruct the chat state without parsing the entire markdown file.

```json
{
  "id": "chat-id-123",
  "userId": "user-id-456",
  "title": "Conversation Title",
  "workflowId": "optional-workflow-id-789",
  "createdAt": "2026-04-12T10:00:00.000Z",
  "updatedAt": "2026-04-12T10:05:00.000Z",
  "messageCount": 12,
  "tags": ["workflow-generation", "image-editing"]
}
```

### `index.md`

The `index.md` file contains the actual conversation. It uses a specific formatting convention to ensure that messages can be easily parsed back into structured data while remaining perfectly readable in any markdown viewer.

#### Header
The document must begin with a level-1 heading containing the chat title, followed by an optional blockquote containing key metadata.

```markdown
# Chat: Conversation Title
> ID: chat-id-123
> Created: 2026-04-12 10:00:00 | Updated: 2026-04-12 10:05:00
```

#### Messages
Each message is separated by a level-2 heading indicating the sender's role (`User`, `Assistant`, or `System`) and the timestamp of the message.

```markdown
## [User] 2026-04-12 10:01:00

I need to create a workflow that takes an image, upscales it, and then turns it into a video.

## [Assistant] 2026-04-12 10:01:15

I can help with that. You'll need three nodes for this pipeline:
1.  An **Image** input node.
2.  A **Precision Upscale** node (or Creative Upscale).
3.  A **Kling 3 Video** generation node.

Here is the JSON representation of that workflow:

\`\`\`json
{
  "nodes": [
    // ... node data ...
  ],
  "edges": [
    // ... edge data ...
  ]
}
\`\`\`
```

#### Attachments
When a message includes or references an attachment (e.g., an image generated by the AI), it should be referenced using a relative path to the `attachments/` directory.

```markdown
## [Assistant] 2026-04-12 10:05:00

Here is the upscaled image you requested:

![Upscaled Output](attachments/upscaled-result.png)
```

## Parsing Rules

When reading `index.md` to reconstruct the chat state in the UI:
1.  **Split by Headers**: Use regex to split the document by `## [Role] Timestamp` headers.
2.  **Extract Role**: The text inside the brackets `[]` determines the `role` property (`user`, `assistant`, `system`).
3.  **Extract Content**: The content following the header until the next `##` or EOF is the message content.
4.  **Resolve Assets**: Ensure relative paths in markdown images (`![alt](path)`) or links are resolved correctly relative to the `chats/{userId}/{chatId}/` directory.
